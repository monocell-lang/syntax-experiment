* The scope of a variable tells us where its declared and destroyed  
* When it goes outa scope, its ded😵. Using a destroyed obj = UB city👻

## # Modules  
* Modules scope up symbols including function, struct, enum, and static variabls   
* Compiler will scream at you if you put main() in a scope 😖!  
  
### Access policies  
* **pub** (public) means the symbol can be used outside the module but still considered inside the module's scope  
* **priv** (private) means the symbol can not be used outside the module. The default policy if you aint explicit  
* Can be used as field or mark on each symbol  
  
Example 1: modules  
```
module A
{
   int helper() {...}
   pub int func() {...}
}

   println("func() = { A::func() }");
   //✅ A::func is visible

   println("helper() = { A::helper() }");
   //❌ A::helper is hidden
}

```

```

   int func() {...}    // pub

```

   pub void ovrA() {...}   // pub

   int func() {...}        // pub

### Constant functions  
* Some functions are qualified to be called at compile time. These are called const-qualified functions  
* When these functions are invoked, they become constant expressions (even if the function returns nothing)  
* Only contant variables can be used as const-qualified functions  
* Const qualification is automatic but you can use [const] to let the compiler check whether it is or nah  
  
📔 Basic principle of compile time evaluation  
1. The data must be available at compile time  
2. Evaluating the expression mustn't change the outside world   
3. Evaluated result mustn't depend on its environment  
  
🪴 To achieve const qualification, the function:  
🌿 No reading from terminal or writing to files. Writing to terminal or reading from files is ok  
👉 Print compilation logs or using files as program configuration without hardcoding things in the source code  
🌿 No mutable static variables (using address is ok tho)  
👉 It creates some surprises in your code 🐲  
🌿 Must not access any out-of-bound addresses  
👉 Memory outside declared variable territory = 🌀  
🌿 No leaking virtual stack or heap pointer in any branches  
👉 The data those pointer pointed to is already ded😵 when the program actually runs  
🌿 No bit reinterpretation between a pointer and a non-pointer  
👉 Not gonna let you hide a pointer. Ble ble 😛👻

[const] double sqrt(double mut x)
{
   if (x < 0)  return NaN;
   if (x == 0) return 0
   if (x == 1) return 1;

   double mut scale = 1;
   while (x < 0.4)
   {
      x *= 4;
      scale *= 0.5;
   }

   while (x > 1.6)
   {
      x *= 0.25;
      scale *= 2;
   }

   double mut y = 0.5 * (x + 1);
   for (0..4)
   {
      y = 0.5 * (y + x / y);
   }

   return y * scale;
}

var1 <- var2 <- var3 <- ... <- (var1)
